# Week 3 – Network Intrusion Detection (Snort)

## Lab: Intrusion Detection with Snort

### Objective
Detect and analyze malicious network activity using a Network Intrusion Detection System (NIDS).

### Tools Used
- Snort (IDS)
- Docker (pytbull-ng attack simulation)
- Linux CLI
- Vim

### What I Did
- Installed and configured Snort on a Linux virtual machine  
- Used Docker to simulate real-time network attacks with pytbull-ng  
- Monitored live network traffic to observe attack behavior  
- Wrote custom Snort rules to detect and block specific traffic  
- Tested IDS rules against simulated attack scenarios  

---

## Project: Network Attack Detection & Prevention

### Objective
Identify and prevent malicious network activity using rule-based intrusion detection.

### What I Did
- Analyzed attack traffic generated by a simulated attacker  
- Created custom Snort rules to detect suspicious patterns  
- Configured Snort to monitor and alert on malicious activity  
- Evaluated effectiveness of IDS rules in blocking unwanted traffic  

---

## Key Concepts
- Network Intrusion Detection Systems (NIDS)  
- Signature-based detection  
- Packet inspection  
- Attack simulation  
- Rule-based detection  

## Key Learning
This lab helped me understand how intrusion detection systems work in real-world scenarios and how to create rules to identify and respond to malicious network behavior.
