# Week 5 – Network Traffic Interception & DoS Analysis

## Lab: Man-in-the-Middle (MITM) & Traffic Analysis

### Objective
Analyze encrypted network traffic and understand how attackers can intercept and modify communication using a proxy.

### Tools Used
- Wireshark
- mitmproxy
- Docker
- Linux CLI

### What I Did
- Captured and analyzed encrypted HTTPS traffic using Wireshark  
- Installed and configured mitmproxy to intercept network traffic  
- Set up a proxy environment to view and modify encrypted requests  
- Compared encrypted vs intercepted traffic to understand differences  
- Observed how attackers can manipulate communication in transit  

---

## Project: DoS Attack Analysis & Mitigation

### Objective
Understand how Denial of Service (DoS) attacks impact systems and how to detect and mitigate them.

### What I Did
- Observed DoS attack behavior through network traffic analysis  
- Identified patterns such as high request rates and abnormal traffic spikes  
- Analyzed server response under attack conditions  
- Implemented mitigation strategies using server configurations (e.g., Nginx)  
- Compared system behavior before and after applying protections  

---

## Key Concepts
- Man-in-the-Middle (MITM) attacks  
- HTTPS traffic analysis  
- Proxy-based interception  
- Denial of Service (DoS / DDoS)  
- Traffic pattern analysis  

## Key Learning
This lab helped me understand how attackers intercept and manipulate network traffic and how abnormal traffic patterns can indicate denial-of-service attacks, as well as how to apply basic mitigation strategies.
