# Week 6 – SIEM Analysis (Splunk)

## Lab: Security Analysis with Splunk

### Objective
Use a SIEM tool to search, analyze, and visualize log data in order to identify potential security incidents.

### Tools Used
- Splunk (SIEM)
- Linux CLI
- Web interface (Splunk dashboard)

### What I Did
- Accessed and navigated the Splunk environment and interface  
- Built and executed search queries using Splunk syntax  
- Filtered logs using fields such as index, source, host, and time  
- Analyzed authentication logs to identify suspicious login activity  
- Created dashboards and visualizations to better understand data patterns

## Project: Log Investigation & Threat Detection

### Objective
Identify suspicious activity and potential threats by analyzing large datasets within a SIEM platform.

### What I Did
- Investigated log data to detect anomalies and unusual patterns  
- Correlated events across multiple data sources  
- Identified indicators of compromise (IoCs) within log data  
- Used Splunk queries to narrow down suspicious behavior  
- Applied data analysis techniques to support security investigations

## Key Concepts
- Security Information and Event Management (SIEM)  
- Log aggregation and correlation  
- Threat detection and monitoring  
- Data visualization and dashboards  
- Indicators of Compromise (IoCs)  

## Key Learning
This lab helped me understand how SIEM tools like Splunk are used in real-world SOC environments to monitor, analyze, and investigate security events at scale.
